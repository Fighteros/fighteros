## 👋 About

I'm a backend-leaning full-stack engineer with 5+ years of experience building production systems that handle real traffic. My work sits at the intersection of **API design, distributed systems, and developer experience** — I care about the boring things that keep services fast, observable, and cheap to operate at 3 AM.

I've shipped platforms for awards management, job aggregation, and SaaS tooling, and I maintain open-source packages on npm (`@nestops/create-backend`, `filemodule`). I think in terms of **contracts, latency budgets, and failure modes** before I think in frameworks.

- 🏗️ &nbsp;**Currently designing:** service boundaries, caching layers, and event-driven workflows in NestJS
- 🧭 &nbsp;**Thinking about:** Postgres internals, Redis data structures, and Git as infrastructure for AI agent workflows
- 📚 &nbsp;**Always sharpening:** system design, distributed consensus, and the cost of abstractions
- 💬 &nbsp;**Happy to talk about:** API versioning, idempotency, queue semantics, or why your ORM is lying to you

## 🧱 Engineering Focus

| Area | What that means in practice |
|---|---|
| **Architecture** | Service decomposition, REST & event-driven patterns, bounded contexts, API contracts that don't break clients |
| **Data** | Schema design, query planning, indexing strategy, Redis as a first-class primitive (not just a cache) |
| **Reliability** | Idempotency, retries with backoff, circuit breakers, observability before features |
| **Code quality** | SOLID where it pays rent, DRY where it doesn't hurt, tests that catch regressions instead of mocking reality |
| **DX** | Clean module boundaries, fast CI, documentation that treats the reader as an engineer |
